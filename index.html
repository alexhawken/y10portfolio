Term 2

Arduino GPT Fact Machine

Create a unique, functional Arduino Project.

Project Overview

An interactive Arduino project that combines ChatGPT with a LCD display to create an engaging fact-sharing device.

Technologies Used

• Arduino Uno WIFI
• LCD Display
• 10k Potentiometer
• 220 ohm Resistors
• Green LED
• Button
• Breadboard & Jumper Wires

Key Features

• Random fact generation and display
• Interactive button controls
• User-friendly interface
• Easy to set-up

Evaluation

Overall, I found this project very interesting and complicated, this was expected since I was doing an AI intergrated project. However, this was made more challenging as I didn’t expect the Arduino libraries I used to be broken. Once my project was done, I found it harder to spot issues, which was why my button broke further into my project rather than earlier. If I were to do it again, I wish I used a bigger screen and another AI model, I found out that ChatGPT isn’t that smart in these situations so it’s better to use a web search based model or something similar. Lastly, I had a lot of fun while building project, I enjoyed being able to make my own project and found the challenges throughout interesting and fun.

Term 3

3D Printed Model

3D modeling & 3D printing project

Project Overview

A creative 3D project involving creating a cat shaped cover for a mouse.

Technologies Used

• Blender 3D software
• Boolean Function
• Seperation Technique
• Merge Function

Creative Process

• Cat design and modeling
• 3D Printing Exporting
• Trial & Testing
• Addressing Issues
• Final output

Evaluation

Overall, my work that I’ve achieved so far has made progress and is almost at completion. I combined my mouse and cat model to create a mouse cat, which in my first trial had a couple issues. The hardest issue was probably creating the ears, as these had to fit inside the ear cutouts and be long enough to reach the mouse. In my first attempt to print my model I learnt that there were no supports, causing my model to ‘spaghettify’ when printing, luckily this was realised quickly and was reprinted. Once printed, In my first trial the ears couldn’t reach the mouses buttons and the left ear couldn’t fit inside the holes, which taught me a lot for my next version. Which, I lengthened the ears, and made smaller, which fixed the main issue with my cat mouse not functioning. I then printed just the ears; however, I had the same issue with my left ear, not fitting. The good news however, I made progress as the right ear could now click the mouses buttons. Sadly, another issue become more predominant, which was the mouse couldn’t fit inside the cat. After I got feedback my latest model came into place, I used what I had learnt to create more space inside the cat, while shrinking the cat's footprint (space taken up). I also edited the ears to properly fit, leaving me with the version I have today.

If I had more time, I would print my new version and test it, and if any more issues would arise (i.e. ears not reaching the buttons, ears come out of the hole, etc.), I would fix those. I would also add a scroll wheel, however with the time and knowledge restraints I had I wasn’t able to implement this.

Term 4

Final End Of Year Portfolio

Webpage Version 2

Project Overview

To create a personal digital presence that represents who I am, my skills, and the projects I've worked on in digital technology this year.

Technologies Used

• Claude AI
• Visual Studio Code

Process of Development

Claude AI was used to make the majority of my End Of Year portfolio page. After multiple itterations I ended up with the website you're currently on.

My first website itteration involved a similarly styled base, however it was a lot more clunky, where each different project took up a seperate page, removing the sense of progression and the time-scale. My second version pivoted in the opposite direction, where as you completed a maze, more aspects of this portfolio was unlocked, however this didn't work as after some feedback and consideration, portfolios are designed to show off your work, not lock it behind levels. This is where I ended up with this website design, getting inspiration from websites such as Whittaker's where they create a unique, animated timeline page, which was something I wanted my site to be similar to. I then began making my new website with all this in mind. My first new version used a dark colour scheme, which after consideration and looking at the contrast values I found it was hard to read, this was where I returned back to a lighter-varient, with red as my primary colour. I also wanted my website to be similar to Raycast's, as they use clean animations and have an easy to understand design layout. This was something I told Claude I wanted mine to be similar to.

Once I had reached my final design the most challenging part was adding additional features, as Claude would give incorrect responses, bugs, etc. This became even more challenging once my token limit would run out, limiting the amount of progress I could make each day. To combat this I had to edit parts of the code in Visual Studio, for example there was one bug where Claude left unformatted, random html code at the bottom of the webpage, requiring me to manually delete it.

You can see some previous design choices, and issues I encoutered to the side.

Main Issues/Bugs

• Creating the loading screen was rather difficult, as the certain style I wanted was hard to describe to Claude, meaning I had to go through multiple alterations until I reached the current design.
• Making the transition between each page, as Claude didn't add a debounce, making it easily breakable. It also took a while to find a design I liked.
• Customising the timeline bar on the side, as it would take multiple attempts to get this working as planned.

Reflecting on my digital technology journey

Final Portfolio Evaluation

Overall, I found this to be quite an enjoying task, although not as challenging as previous assessments this webpage allowed me to make a professional looking portfolio, and I believe my outcome reflects that. The tools I used (Claude AI) allowed me to create my vision for the perfect portfolio page. Reflecting back onto the steps that brought me here, I feel I should've managed my time better, as once I was out of Claude AI tokens my progress grinded to a halt. If I were to do it again I would've liked to seen more animations and gradients being incorporated, however the limit on AI tokens and the timeframe prevented this. I would've also liked to see some way I could've incorperated my maze type portfolio into my final version, without compromising on information shared.

Final Synthesis

I've found this year to be quite enjoying overall. Each topic has widened my knowledge and has been challenging, yet educating. My personal favourite would be my ChatGPT Arduino project, as I found it quite challenging (yet fun!) to overcome each obstacle thrown at me, whether the broken libaries, or the incorrect Arduino module, each obstacle was unforeseen and satisfactory to overcome. The one project I wanted to re-do was my portfolio page, I personally disliked the design, and the functionality of my original portfolio, so I'm glad I got to remake it into this.
